# Tricky Parts of JavaScript 🚀

A comprehensive collection of JavaScript learning resources focused on advanced and tricky concepts that every JavaScript developer should understand.

## 📚 Topics Covered

### Scope & Hoisting
- **blockscope.js** / **blockscope2.js** - Understanding block scope with `let` and `const` vs function scope with `var`
- **scope.js** / **globalscope.js** / **globalscope2.js** - Exploring different scope types and scope chain
- **scopechallege.js** - Practical challenges to test scope understanding
- **hoisting.js** - Understanding variable and function hoisting behavior

### Functions
- **function.js** - Core function concepts and declarations
- **functionexp.js** / **functionexp2.js** - Function expressions and arrow functions
- **functionchal.js** - Function challenges and practice problems
- **IIFE's.js** / **IIFE'S2.js** / **IIFE's3.js** - Immediately Invoked Function Expressions

### Asynchronous Programming
- **promise.js** / **promise2.js** - Understanding Promises and promise chaining
- **promisecha.js** - Promise challenges and practical scenarios
- **promise.all.js** - Working with Promise.all() for concurrent operations
- **async.js** - Async/await syntax and patterns
- **fetch.js** / **fetch2.js** - Making HTTP requests with the Fetch API

## 💡 Learning Tips

1. **Understand the fundamentals**: Start with scope and hoisting before moving to asynchronous concepts
2. **Practice challenges**: Each challenge file helps reinforce learning
3. **Experiment**: Modify code and see how it behaves
4. **Use browser console**: Open the HTML file to inspect console output
5. **Trace execution**: Use `console.log()` statements to track code execution flow

## 🔍 Common Pitfalls to Avoid

- Confusing `var` (function-scoped) with `let`/`const` (block-scoped)
- Missing the temporal dead zone (TDZ) with `let` and `const`
- Not understanding promise states and rejection handling
- Improper error handling in async/await
- Forgetting that `async` functions return promises
